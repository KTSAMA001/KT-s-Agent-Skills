# 查找流程

> 用户遇到问题时，从已有记录中检索

---

## 一、查找步骤

1. **判断问题领域** → 参考 [INDEX.md](../INDEX.md) 确定检索分类
2. **读取相关文件** → 读取对应 `data/[experiences|knowledge]/<分类>/*.md`
3. **关键词匹配** → 搜索标题、标签、内容
4. **呈现结果**：
   - ✅ 找到 → 展示内容，询问是否适用
   - ❌ 未找到 → 转入 **网络搜索**

---

## 二、搜索技巧

| 技巧 | 说明 |
|------|------|
| **关键词** | 使用核心概念，如 "协程"、"yield" (非整句) |
| **标签过滤** | 利用 `#tag` (如 `#unity2021`) 快速筛选 |
| **状态优先** | 优先采纳 ✅ 已验证 / 📘 有效的内容 |
| **版本检查** | **关键**：确认记录版本与当前环境匹配 (如 Unity 2021 vs 6000) |

---

## 三、网络搜索（本地未找到时）

**步骤**：

1. 告知用户："未在记录中找到，正在搜索..."
2. 构建搜索词 → 提取核心关键词
3. 执行搜索 → 使用 可用的网络搜索工具

**搜索来源优先级**：

| 优先级 | 来源 |
|--------|------|
| 1 | 官方文档（Unity Docs、Microsoft Docs、MDN） |
| 2 | 官方论坛（Unity Forum、GitHub Discussions） |
| 3 | Stack Overflow（高票答案优先） |
| 4 | 技术博客（知名作者、详细解析） |

**结果评估**：

| 检查项 | 说明 |
|--------|------|
| 时效性 | 内容发布日期，API 是否已变更 |
| 权威性 | 官方 > 社区 > 个人 |
| 完整性 | 是否有完整示例代码 |

**后续**：解决问题后询问是否记录 → 转入 [record.md](./record.md)
