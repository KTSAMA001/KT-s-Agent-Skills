# 查找流程

> 用户遇到问题时，从已有记录中检索

---

## 一、查找步骤

1. **提取关键词** → 从用户问题中提取核心概念
2. **查索引** → 读取 [INDEX.md](../INDEX.md)，按标签或文件清单定位
3. **读取记录** → 读取 `data/*.md` 对应文件
4. **呈现结果**：
   - ✅ 找到 → 展示内容，询问是否适用
   - ❌ 未找到 → 转入 **网络搜索**

---

## 二、搜索策略

### 按标签检索（首选）
在 INDEX.md 的文件清单表格中搜索标签列：
- 用户问 Unity 渲染 → 在标签列搜索 `#unity`、`#shader`、`#rendering`
- 用户问 Git 问题 → 在标签列搜索 `#git`
- 可先查 **标签概览** 确认有哪些可用标签

### 按关键词检索
在文件清单的简述列搜索，或直接在 `data/` 下 grep：
- 搜索核心概念："协程"、"yield"、"CBUFFER"（非整句）
- 利用文件名中的关键词快速匹配

### 搜索技巧

| 技巧 | 说明 |
|------|------|
| **标签过滤** | 在文件清单标签列搜索 `#tag` |
| **关键词** | 使用核心概念而非整句 |
| **状态优先** | 优先采纳 ✅ 已验证 / 📘 有效的内容 |
| **版本检查** | **关键**：确认记录版本与当前环境匹配 |

---

## 三、网络搜索（本地未找到时）

**步骤**：

1. **告知用户**："未在记录中找到，正在搜索..."
2. **构建搜索词** → 提取核心关键词
3. **执行搜索** → 使用可用的网络搜索工具

**搜索来源优先级**：

| 优先级 | 来源 |
|--------|------|
| 1 | 官方文档（Unity Docs、Microsoft Docs、MDN） |
| 2 | 官方论坛（Unity Forum、GitHub Discussions） |
| 3 | Stack Overflow（高票答案优先） |
| 4 | 技术博客（知名作者、详细解析） |

**结果评估**：

| 检查项 | 说明 |
|--------|------|
| 时效性 | 内容发布日期，API 是否已变更 |
| 权威性 | 官方 > 社区 > 个人 |
| 完整性 | 是否有完整示例代码 |

**后续**：解决问题后询问是否记录 → 转入 [record.md](./record.md)
