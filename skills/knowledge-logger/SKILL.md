# Knowledge Logger 技能

当用户学习到新知识、理解了某个概念、或需要记录技术原理时，使用此技能将知识点记录到对应的文档中。

## 触发条件

当用户说出以下类似内容时触发：
- "记一下这个知识点"
- "这个概念/原理记录下来"
- "把这个定义保存一下"
- "学到了，记下来"
- 讨论完某个技术原理后需要归档

## 知识库位置

`~/.copilot/knowledge/`

## 知识条目格式

```markdown
## [知识点标题]

**分类**：[一级分类] > [二级分类]
**关键词**：#关键词1 #关键词2
**来源**：[书籍/文档/课程名称]
**可信度**：[评级标注]

### 定义/概念

[简明扼要的定义或概念描述]

### 原理/详解

[详细的原理解释，可包含公式、图示说明等]

### 关键点

- 要点 1
- 要点 2
- 要点 3

### 示例（可选）

[代码示例或实际案例]

### 相关知识

- [相关知识点 1](链接)
- [相关知识点 2](链接)

### 与经验关联（可选）

- 相关经验：[experiences/xxx/xxx.md](链接) - 实践验证了此知识点

---
```

## 可信度评级标准

| 评级 | 标识 | 说明 | 示例来源 |
|------|------|------|----------|
| 最高 | ⭐⭐⭐⭐⭐ | 官方规范/权威著作/学术标准 | 官方文档、经典教材、论文 |
| 高 | ⭐⭐⭐⭐ | 官方文档/实践验证/厂商文档 | Unity 博客、GPU 厂商文档 |
| 中 | ⭐⭐⭐ | 社区公认/广泛实践验证 | 知名博客、技术分享、开源项目 |
| 低 | ⭐⭐ | 个人经验/小范围验证 | 个人笔记、论坛讨论 |
| 待验证 | ⭐ | 来源不明/待验证 | 未经验证的内容 |

## 执行步骤

1. **识别知识类型**：判断属于哪个分类（图形学/Unity/编程/HLSL/工具等）
2. **确定目标文件**：根据具体主题选择或创建对应的 md 文件
3. **检查是否已存在**：避免重复记录相同知识点
4. **格式化知识条目**：按照标准格式整理知识点
5. **追加到文档**：将知识条目追加到目标文件末尾
6. **关联经验**（可选）：如果有相关实践经验，添加相互引用

## 知识与经验的区别

| 维度 | 知识 (Knowledge) | 经验 (Experience) |
|------|------------------|-------------------|
| 来源 | 书籍、文档、课程、规范 | 实际开发、踩坑、调试 |
| 格式 | 概念 → 原理 → 要点 | 问题 → 解决方案 → 验证 |
| 目的 | 理解"是什么"和"为什么" | 记录"怎么做"和"怎么避坑" |
| 时效性 | 相对稳定 | 可能随版本变化 |
| 验证 | 理论正确性 | 实践有效性 |

## 相互引用示例

**在知识文档中**：
```markdown
### 与经验关联
- 实践验证：[CBUFFER 合批经验](../experiences/shader/urp.md#cbuffer-与-srp-batcher-合批机制)
```

**在经验文档中**：
```markdown
**理论基础**：
- 参考知识：[SRP Batcher 原理](../knowledge/unity/srp.md#srp-batcher)
```

## 注意事项

1. 知识点应该是**通用的、稳定的**技术原理，而非特定问题的解决方案
2. 优先使用**简洁明了**的语言，避免大段复制粘贴
3. 如果知识来源于特定书籍/文档，务必注明来源
4. 与现有经验有关联时，添加相互引用以便日后查阅
